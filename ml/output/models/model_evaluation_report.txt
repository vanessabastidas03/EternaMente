7. LIMITACIONES Y TRABAJO FUTURO
----------------------------------------------------------------------
  a) Datos sintéticos: el modelo se entrenó con datos generados por
     generate_synthetic_data.py, NO con datos clínicos reales.
     Para despliegue en producción se requiere validación con datos reales.

  b) Fuga temporal: el split train/test por fila (user × semana)
     puede tener fugas entre semanas del mismo usuario. Para evaluación
     definitiva usar GroupShuffleSplit por user_id.

  c) Desbalance de clases: NORMAL=93%, WATCH=6%, ALERT=1%.
     El peso 'balanced' mitiga el problema pero métricas macro/micro
     pueden sobre/sub-estimar el rendimiento en clases minoritarias.

  d) Horizonte temporal: el modelo asume 4 semanas de datos pasados.
     Para usuarios con < 4 semanas de historial la fiabilidad es menor.
